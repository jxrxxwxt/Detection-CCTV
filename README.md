# ระบบตรวจจับและตรวจสอบรถบรรทุกผ่าน CCTV (Truck Inspection Detection System)

เอกสารนี้อธิบายรายละเอียดสถาปัตยกรรม ลอจิกการทำงาน และวิธีการใช้งานระบบ AI ตรวจจับรถบรรทุกและพนักงานรักษาความปลอดภัย (รปภ.) จากวิดีโอ CCTV เพื่อตรวจสอบว่ารถบรรทุกที่ไม่มีของ (Unloaded Truck) ได้รับการตรวจสอบอย่างถูกต้องหรือไม่

## 📌 สรุปภาพรวมของโปรเจกต์ (Project Overview)

ระบบนี้ใช้โมเดล AI (YOLOv8) ในการวิเคราะห์วิดีโอ CCTV แบบเฟรมต่อเฟรม (Frame-by-Frame) โดยมีเป้าหมายหลักคือ:
1.  **ติดตาม (Tracking):** ติดตามรถบรรทุกแต่ละคันที่เข้ามาในพื้นที่
2.  **วิเคราะห์สถานะ (State Analysis):** คำนวณความเร็วเพื่อดูว่ารถ "กำลังเคลื่อนที่" หรือ "จอดสนิท"
3.  **ตรวจสอบเงื่อนไข (Inspection Validation):** ตรวจสอบว่า รปภ. ได้เดินเข้าไปใกล้รถบรรทุก (ที่จอดสนิท) มากพอที่จะถือว่า "ทำการตรวจสอบแล้ว" หรือไม่
4.  **แจ้งเตือน (Alert System):** แจ้งเตือนทันทีหากรถบรรทุกขับออกไปโดยที่ยังไม่ได้รับการตรวจสอบ
5.  **บันทึกผล (Logging & Export):** แสดงสถานะของรถทุกคันแบบ Real-time บนหน้าจอ และบันทึกเป็นไฟล์วิดีโอที่มีความไหลลื่น 100%

## 🧠 เจาะลึกตรรกะการทำงาน (Deep Dive into Logic)

### 1. การคำนวณความเร็วและสถานะหยุดนิ่ง (Speed & Stop Detection)
เพื่อป้องกันปัญหา "กรอบสั่น (Bounding Box Jitter)" ที่ทำให้ AI เข้าใจผิดว่ารถกำลังวิ่งแม้จอดนิ่ง ระบบจึงใช้ 3 เทคนิค:
*   **Bottom-Center Tracking:** วัดระยะจาก "จุดกึ่งกลางของฐานล้อรถ" (ขอบล่างของกรอบ) แทนที่จะเป็นตรงกลางรถ เพราะฐานล้อจะนิ่งที่สุดติดกับพื้นถนน
*   **Moving Average (30 Frames):** ระบบจะจำตำแหน่งย้อนหลังของรถแต่ละคันไว้ 30 เฟรม (ประมาณ 1 วินาที) แล้วหาระยะขจัดรวมเพื่อหา "ความเร็วเฉลี่ย (`stable_speed`)" วิธีนี้ช่วยกรอง Noise ออกไปได้อย่างหมดจด
*   **Strict Threshold & Grace Period:**
    *   ความเร็วต้องต่ำกว่า `1.0 พิกเซล/เฟรม` ถึงจะถือว่าจอดสนิท
    *   มีช่วงเวลาผ่อนผัน (`GRACE_PERIOD = 15 เฟรม`) เมื่อรถโผล่มาครั้งแรก ระบบจะไม่รีบตัดสินใจจนกว่า Tracker จะจับภาพได้เสถียร (แสดงสถานะ `DETECTING...`)

### 2. การตรวจสอบการทำงานของ รปภ. (IoU Overlap Percentage)
ระบบไม่ได้เช็คแค่ว่ากรอบของ รปภ. "แตะ" กับรถหรือไม่ แต่ใช้สูตรคณิตศาสตร์เพื่อคำนวณสัดส่วนพื้นที่:
*   **ฟังก์ชัน `calculate_overlap_percent`:** คำนวณหาพื้นที่ทับซ้อนกันระหว่างกรอบ รปภ. กับ กรอบรถ
*   จากนั้นนำพื้นที่ทับซ้อนมาหารด้วย "พื้นที่ทั้งหมดของ รปภ." จะได้เป็นค่าเปอร์เซ็นต์ (0% - 100%)
*   **เงื่อนไขผ่าน (`>= 30%`):** ระบบจะยอมรับว่า "ตรวจแล้ว" ก็ต่อเมื่อตัว รปภ. เข้าไปซ้อนทับกับรถบรรทุกอย่างน้อย 30% ของร่างกาย (ป้องกันการเดินผ่านด้านหลังไกลๆ แล้วระบบตีความผิด)

### 3. ระบบ State Machine และ State Lock (การจัดการสถานะ)
รถบรรทุก (Unloaded) แต่ละคันจะมีสถานะวนลูปดังนี้:
1.  **`MOVING (Incoming)`:** รถกำลังวิ่งเข้ามา
2.  **`STOPPED (Waiting Guard...)`:** รถจอดสนิทแล้ว รอ รปภ. เดินมาใกล้ (สีเหลือง)
3.  **`PASSED (Inspected)`:** รปภ. เดินมาใกล้เกิน 30% ในขณะที่รถจอดนิ่ง ถือว่าผ่าน (สีเขียว)
4.  **`ALERT! (Missed Inspection)`:** รถเคยจอดแล้ว แต่ขับออกไป (ความเร็วพุ่งสูง) โดยที่ยังไม่ได้เป็นสถานะ PASSED (สีแดง)

**🔴 State Lock Feature:** หากสถานะเปลี่ยนเป็น `ALERT!` แล้ว โค้ดจะทำการ "ล็อค" สถานะนั้นไว้ถาวรสำหรับ ID รถคันนั้น ต่อให้รถกลับมาจอดใหม่ สถานะก็จะยังคงแจ้งเตือนภัยอยู่ เพื่อใช้เป็นหลักฐานการทำผิดกฎ

## 🎨 สถาปัตยกรรม UI (User Interface Architecture)

เพื่อไม่ให้ข้อความไปบดบังภาพจากกล้องวงจรปิด โค้ดถูกออกแบบให้ขยายเฟรมวิดีโอ (Extended Canvas):
*   **สร้าง Canvas ใหม่:** ระบบสร้างภาพพื้นหลังสีดำว่างๆ ที่มีความกว้างเท่ากับวิดีโอเดิม + 500 พิกเซล
*   **วิดีโอหลัก (ขวา):** วางวิดีโอที่ AI วิเคราะห์แล้วไว้ทางขวา
*   **Log Panel (ซ้าย):** ใช้พื้นที่ 500 พิกเซลทางซ้ายเพื่อแสดง **CCTV TRUCK LOGS**
    *   ลิสต์ Track ID ของรถแต่ละคันเรียงลงมา
    *   แสดงสถานะล่าสุดของรถคันนั้น พร้อมสีที่บ่งบอกความปลอดภัย (เขียว/เหลือง/แดง)
*   **High-Contrast Overlap Text:** ข้อความที่แสดงเปอร์เซ็นต์ Overlap บนหัว รปภ. จะถูกวาดทับบนกรอบสีดำทึบ ทำให้มองเห็นได้ชัดเจนทะลุฉากหลัง

## 🚀 วิธีการใช้งาน (How to Run)

### 1. ความต้องการของระบบ (Prerequisites)
ตรวจสอบให้แน่ใจว่าติดตั้ง Library เหล่านี้แล้ว (ติดตั้งผ่าน `pip install -r requirements.txt` หรือพิมพ์ตรงๆ):
```bash
pip install opencv-python ultralytics numpy
```
*(หากเครื่องมีการ์ดจอ NVIDIA แนะนำให้ติดตั้ง PyTorch เวอร์ชัน CUDA เพื่อความเร็วสูงสุด)*

### 2. การรันโปรแกรม
เปิด Terminal/Command Prompt ชี้ไปที่โฟลเดอร์โปรเจกต์ แล้วรันคำสั่ง:
```bash
python detect.py
```

### 3. ผลลัพธ์ระหว่างการทำงาน
*   หน้าต่าง **"YOLO Detection"** จะเด้งขึ้นมาเพื่อแสดงความคืบหน้า (อาจจะดูกระตุกเล็กน้อยเพราะเป็นการจำลองภาพแบบสดๆ บน CPU/RAM)
*   คุณสามารถกดปุ่ม **`q`** บนคีย์บอร์ดเพื่อหยุดการทำงานกลางคันได้
*   เมื่อโปรแกรมทำงานจนจบวิดีโอ หรือถูกกดหยุด จะมีข้อความแสดงใน Console ว่า `Detection completed.`

### 4. การดูไฟล์ผลลัพธ์ (The Final Output)
เพื่อประสบการณ์ที่ดีที่สุดและความไหลลื่น 100% ให้เปิดไฟล์:
👉 **`output_with_log.mp4`**
